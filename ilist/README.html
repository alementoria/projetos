01 · o produto

O que é o DyList

DyList é um player web para YouTube que coloca o controle de volta nas suas mãos. Você monta suas playlists por categorias, reproduz sem interrupções e o algoritmo não interfere. Funciona como PWA — instala no celular e roda offline com cache local.

É um projeto de portfólio pessoal, disponibilizado gratuitamente. Não é um produto comercial — construído para demonstrar capacidade técnica e de produto, não para monetizar.

02 · arquitetura

Como o código está organizado

Cada responsabilidade vive em seu próprio módulo. Não há framework — JavaScript vanilla com separação de concerns deliberada. A decisão foi manter zero dependências de build para que qualquer pessoa possa abrir, ler e modificar sem toolchain.

app.js

Núcleo da aplicação. Estado global da playlist, renderização, drag & drop, ações (deletar, selecionar, limpar). Ponto de entrada que orquestra os outros módulos.

player.js

Wrapper da YouTube IFrame API. Controla play, pause, skip, velocidade, loop, Media Session API e atalhos de teclado. Ponto de risco principal — dependência da API do YouTube.

categories.js

Sistema de categorias. CRUD completo, drag & drop para reordenar, emoji picker, persistência no localStorage. Independente do player.

search-advanced.js

Input unificado. Detecta tipo de entrada (vídeo, canal, playlist, busca), consome YouTube Data API v3, paginação, modal de resultados com seleção em lote.

backup.js

Sistema de backup. Exporta e importa playlist + categorias + configurações em JSON. Auto-backup periódico e antes de fechar a aba.

equalizer.js

Equalizador 8 bandas. Usa Web Audio API para capturar e processar o áudio. Modos visual (espectro animado) e processamento real quando o contexto permite.

sidebar.js

Sidebar de tutoriais. Menu lateral com vídeos tutoriais, formulário de API key e atalho de teclado Ctrl+H.

onboarding.js

Balloons de onboarding. 4 passos sequenciais no primeiro acesso. Overlay SVG com recorte iluminado, seta direcional, dots de progresso. Salva conclusão no localStorage.

pwa.js

PWA + Service Worker. Registro via Blob URL, captura do prompt de instalação, wake lock e verificação de atualizações periódica.

ad-blocker.js

Bloqueio de anúncios externos. CSS blocking + DOM observer para elementos de propaganda fora do player. SponsorBlock-like para segmentos marcados manualmente.

03 · decisões

Por que foi feito assim

Todo projeto acumula decisões. Essas são as que mais importam para entender o código.

Sem framework, sem build

JavaScript vanilla com módulos carregados via <script src> sequenciais. Qualquer editor de texto abre, qualquer servidor estático serve. Complexidade zero de toolchain foi uma escolha deliberada para um produto que precisa ser fácil de modificar e hospedar.

localStorage como única persistência

Todos os dados vivem no navegador. Isso significa que trocar de dispositivo ou limpar o cache apaga tudo — o backup manual em JSON existe exatamente por isso. A decisão foi consciente: adicionar backend adicionaria complexidade de infraestrutura e conta de usuário, o que não faz sentido para um projeto gratuito sem autenticação.

Dependência total do YouTube

O player depende inteiramente da YouTube IFrame API. Isso é um risco existencial documentado: se o YouTube mudar a API ou bloquear embeds, o player para de funcionar. A mitigação correta seria criar um YouTubeAdapter isolado, facilitando a troca de plataforma. Essa refatoração está mapeada mas ainda não foi feita.

Sem monetização por razões legais

O DyList usa a YouTube IFrame API em contexto que potencialmente viola os Termos de Serviço do YouTube se comercializado. A decisão de distribuir gratuitamente, sem plano pago e sem anúncios, foi tomada conscientemente para manter o projeto dentro de uma zona legal defensável.

05 · próximos passos

O que vem depois

Em ordem de impacto real, não de complexidade técnica.

1 · Abstrair o player (3–4h)

Separar player.js em PlayerCore.js + YouTubeAdapter.js. Não reescreve comportamento — só isola a dependência. Isso transforma um risco existencial em um arquivo trocável.

2 · Suporte a SoundCloud

SoundCloud tem embed estável, API pública generosa e não serve anúncios no widget. Adicionar um SoundCloudAdapter reduziria a dependência de uma única plataforma.

3 · Unificar ad-blocker.js e premium.js

Os dois arquivos fazem a mesma coisa com implementações levemente diferentes. Dívida técnica acumulada que precisa de consolidação antes de crescer mais.

07 · honestidade

O que este projeto não é

✦ nota do autor

DyList não é um produto comercial e não deve ser tratado como tal. O uso da YouTube IFrame API para bloquear anúncios provavelmente viola os Termos de Serviço do YouTube em contexto comercial. Distribuir gratuitamente é uma posição legalmente mais defensável — mas não uma garantia.

O projeto existe como demonstração técnica: prova de que é possível construir uma UX cuidada, código modular e uma experiência de produto real sem framework, sem backend e sem infraestrutura.

Se você quiser usar, modificar ou se inspirar no código — fique à vontade. Se quiser construir algo comercial em cima disso — leia os ToS do YouTube primeiro e tome sua própria decisão consciente.
